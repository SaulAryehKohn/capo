Basic Usage

Visibilities or baseline listings are extracted from the database written by the correlator recieve script. Once oCAPO is pointed at the correct database its internal visibility index can be updated by clicking Update near the top of the page. This update only needs to be performed when the database is changed or the correlator is reconfigured.

The visibility listing is displayed in a grid on the Home view. The listing may be filtered by antenna number, polerization etc using the Filters to the right of the visibility table. Create or modify filters as necessary. Only one is applied at a time. Refresh the page (ctrl-R) to apply the changes to the visibility listing. Select "streaming" mode to automatically update page

Use the trace option to plot logged average values. Loggin status is indicated in the upper-right corner of the dashboard. Activate logging by navigating to
     cd ~/CAPO_online/CAPO_dashboard/
and then starting the update daemon with
     python manage.py update [dbid]
where the dbid is 1 for beestrong and 2 for beetrue. (This interface will be improved in next cycle.)

Click on a visibility in the table to see a plot of the correlator output. Change the properties of the plot using the table at the right. This function hits the database every time it is generated. To monitor a baseline, simply refresh the page.

Add a correlator database by going into the Admin interface and navigating to the Corr DBs listing and clicking add corr db. Database selection is currently limited to those in /var/www
